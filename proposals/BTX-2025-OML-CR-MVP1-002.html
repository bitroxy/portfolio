Critical Transaction Workflows

Workflow 1: User Deposit (On-Chain)

1
User Requests Deposit Address

Flutter App → Backend API: POST /api/v1/wallet/generate-address

2
Unique Wallet Generation

Backend → BitGo API: Generate unique Tron wallet, encrypt keys, store in PostgreSQL

3
User Sends Funds to Address

User transfers USDT from external wallet. Network gas fees paid by user.

4
Webhook Confirmation

Tron Network → BitGo → Backend Webhook: Transaction confirmed, balance updated

5
Auto-Sweep to Pool Wallet

Background Worker: Transfer from user wallet → Company Pool Wallet. Gas paid from Gas Tank.

6
Audit & Notification

Create immutable audit log, update user internal balance, send FCM notification

Workflow 2: Internal Transfer (Off-Chain)

1
Initiate Transfer

User enters recipient (wallet address OR phone number from contacts), amount, passcode

2
Validation & Authorization

Backend validates JWT, passcode. Checks sender balance. Validates recipient exists in DB.

3
Atomic Ledger Update (Database Transaction)

BEGIN TRANSACTION → Debit sender balance → Credit recipient balance → Create Transaction records → COMMIT. Uses pessimistic locking.

4
Audit Trail & Notifications

Log to audit service, send push notifications to both users. Funds remain in pool custody.

Workflow 3: External Withdrawal (On-Chain)

1
User Requests Withdrawal

User enters external Tron address, amount, authenticates with passcode

2
Address Detection & Fee Calculation

System detects address is external (not in DB). Calculate: network gas fee + configurable service fee.

3
Internal Balance Debit

Lock funds by debiting user's internal balance (amount + fees). Create pending withdrawal record.

4
On-Chain Transaction Execution

Background Worker: Retrieve Hot Pool Wallet credentials → BitGo API broadcast transaction → Wait for confirmations

5
Confirmation & Audit

Update withdrawal status, store TX hash, create audit log, send notification with blockchain explorer link

Technical Scope & Deliverables

Complete rebuild aligned with SAD v1.1 architecture

A. Security-by-Design Implementation

Secrets & Configuration Management
  • Remove all hardcoded credentials from codebase
  • Implement AWS Secrets Manager/Vault integration
  • Centralized ConfigService for all environments
  • Rotate all exposed API keys immediately
  • Git pre-commit hooks to prevent secret leaks
Authentication & Authorization
  • Complete JWT signature & claims validation
  • Secure all unprotected API endpoints
  • Middleware enforcement on all protected routes
  • Refresh token mechanism with rotation
Input Validation & Sanitization
  • Zod schema validation on all API endpoints
  • Server-side validation for Server Actions
  • XSS prevention with input sanitization
  • SQL injection prevention (parameterized queries)
  • Rate limiting on all endpoints
Enterprise Logging Infrastructure
  • Replace console.log with structured logging
  • JSON formatted logs for parsing
  • Remove ALL debug logs from production builds
  • PII/sensitive data sanitization in logs
  • Centralized log aggregation
  • Log levels: ERROR, WARN, INFO, DEBUG
Immutable Audit Trail System
  • Design & implement AuditService module
  • Log all authentication events (success/fail)
  • Track all financial transactions with metadata
  • Tamper-proof logging
  • Audit report generation APIs
Financial Precision Implementation
  • Replace ALL Number types with Decimal.js
  • Update PostgreSQL schema to DECIMAL(20,8)
  • Refactor all balance calculation logic
  • Implement proper rounding strategies
Mobile Security Hardening
  • Implement SSL/TLS certificate pinning
  • Configure Dio client with pinned certificates
  • Certificate rotation mechanism
  • Secure local storage encryption
